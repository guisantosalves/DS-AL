macetes — regras práticas

Ignore constantes e termos menores time+space

Big O é comportamento assintótico. Constantes e termos dominados somem.

3n² + 7n + 42  →  O(n²)
100n           →  O(n)
n/2            →  O(n)

Loops aninhados = multiplicar time

Cada nível de loop aninhado multiplica a complexidade.

for i in n:      # O(n)
  for j in n:  # × O(n)
    work()      # = O(n²)

Loops sequenciais → domina o maior time

Loops separados somam, mas o maior engole o menor.

for i in n: ...   # O(n)
for j in n²: ...  # O(n²)
total → O(n²)

Divide por 2 a cada passo = log n time

Espaço de busca cortado pela metade → logaritmo.

while left <= right:
  mid = (left+right)//2
  ...              # O(log n)

Recursão que ramifica = exponencial time

Sem memoization, ramificação 2x a cada chamada = O(2ⁿ).

fib(n-1) + fib(n-2)  → O(2ⁿ)
fib com memo          → O(n)

Space: o que cresce com n? space

Variáveis simples são O(1). Estruturas que crescem com n contam. Stack de recursão também.

arr = [x for x in input] → O(n)
total = 0               → O(1)
merge_sort call stack   → O(log n)

Inputs separados = variáveis separadas time+space

Dois arrays de tamanhos diferentes precisam de variáveis próprias.

# array a (tamanho m), b (tamanho n)
for x in a: ...  # O(m)
for y in b: ...  # O(n)
→ O(m + n), não O(n)

Iterativo ≠ recursivo no space space

Mesma lógica iterativa pode ser O(1) space vs O(n) na recursiva.

fibonacci recursivo → O(n) space
fibonacci iterativo → O(1) space
# (só prev + curr)
exemplos práticos

Two Sum (hashmap)

T: O(n) S: O(n) 
seen = {}
for i, x in enumerate(arr):
  comp = target - x
  if comp in seen:
    return [seen[comp], i]
  seen[x] = i

Binary Search

T: O(log n) S: O(1) 
l, r = 0, len(arr)-1
while l <= r:
  mid = (l + r) // 2
  if arr[mid] == target: ...
  elif arr[mid] < target: l = mid+1
  else: r = mid-1

Merge Sort

T: O(n log n) S: O(n) 
merge_sort(arr):
  mid = len(arr) // 2
  left  = merge_sort(arr[:mid])
  right = merge_sort(arr[mid:])
  return merge(left, right)

Sliding Window

T: O(n) S: O(1) ou O(k) 
l = 0
for r in range(len(s)):
  # expande window com s[r]
  while invalido:
    # encolhe pelo lado l
    l += 1

BFS / DFS

T: O(V+E) S: O(V) 
# V = vértices, E = arestas
queue = deque([start])
visited = {start}
while queue:
  node = queue.popleft()
  for nb in graph[node]: ...

LRU Cache

T: O(1) S: O(capacity) 
# HashMap + Doubly Linked List
get(key):
  move_to_front(key)
  return map[key]
put(key, val):
  if full: evict_lru()
  map[key] = insert_front(val)

Two Pointers

T: O(n) S: O(1) 
# array ordenado, soma = target
l, r = 0, len(arr)-1
while l < r:
  s = arr[l] + arr[r]
  if s == target: return [l, r]
  elif s < target: l += 1
  else: r -= 1

DP — Fibonacci

T: O(n) S: O(1) 
# bottom-up, O(1) space
prev, curr = 0, 1
for _ in range(n-1):
  prev, curr = curr, prev + curr
return curr
